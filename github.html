Github Notları

Temel Terimler

  1. repository : Kısa ismi ile repo, kodlarımızın saklandığı yerdir.
  2. master : Depodaki ana proje sürümüdür
  3. branch : Git ağaç mantuğu ile çalışır ve projedeki geliştirmelerimizi ağaç dalı mantığı ile ayrı dallarda saklayabilir, snrasında master branch de birleştirebiliriz.
  4. merge : branch lerde yapılan işlemlerin master da birleştirilmesidir.
  5. commit : Commit işlemi lokaldeki dosyalarımızı depoya gönderilmek üzere hazırlar.
  6. push : Dosyalarımızı depoya göndermemizi sağlar.
  7. pull : Depodaki dosyalarımızı çekmemizi sağlar.
  8. .gitignore : Git in takip etmesini istemediğimiz dosya ve klasörleri belirttiğimiz dosyadır.

Vs code da projede bir değişiklik yapıldıktan sonra bunu gitub a göndermek için Source Control tabını açtık. Message alanına yapılan değişiklik ile alakalı bir mesaj yazdık. Son olarak Commit butonuna basıp depoya gönderilmek üzere hazırladık ve Sync butonuna push işlemi yaparak değişikliği github a yolladık.

Temel Git Komutları

  1. git add veya git add *, git add index.html : Klasöre eklenen dosyaları gönderilmek üzere sahneye alma(Stage)
  2. git status : klasördeki tüm değişiklikleri göster
  3. git commit -m "ilk commit" : Depoya gönderilmek için commit işlemi yapılır
  4. git commit -a -m "Mesajımız" : Staging e ekle ve commitle
  5. git push : değişiklikleri depoya gönderme komutu
  6. git pull : Uzak depodaki dosyaları lokale indirme
